## Function Calling

> Function Calling 是由 OpenAI 等模型提供商率先推出的一种模型能力。它允许开发者在调用 LLM 时，描述一系列工具或函数的规格（包括函数名、功能描述、参数列表）。然后，模型在生成回复时，会根据用户的提问，自主判断是否需要调用一个或多个函数。如果模型觉得需要调用，它**不会直接执行代码**，而是返回一个结构化的请求，告诉开发者：“你应该调用 `get_weather` 函数，并且传入参数 `location=‘Beijing’`。”
>
> **本质：** 一种将自然语言转化为结构化指令的**接口**。它是模型理解能力的一种延伸，让模型从“只能生成文本”变成了“可以请求执行外部动作”。
>
> —— DeepSeek 解释

个人理解：Function Calling 之所以称为函数调用，是因为在调用工具的时候，需要给这个工具传入相关的参数，而这些参数来自于LLM对用户对话的思考和提取。一旦被模型判断出需要调用一个或多个函数时，就会告诉开发者，开发者会根据模型的指令去执行对应的函数。

关键点：

- 模型**不直接执行**工具，它只是提出调用请求。
- 实际执行工具的是你（开发者）的代码。

Function Calling 就是给模型配了一个“工具箱”和“说明书”，当模型遇到需要动手或查新信息的问题时，它可以告诉你“请帮我从工具箱里拿出 X 工具，用 Y 参数做一下”。这样，模型就能回答那些原本无法回答的实时性、私密性或需要操作的问题了。



## Agent

>一个 Agent 是一个基于 LLM 的自主系统，它被设计用来完成一个特定的目标。为了实现这个目标，Agent 会进行一系列循环操作：
>
>1. **思考（Think）：** 接收用户指令，分析当前状态，规划下一步需要做什么。
>2. **行动（Act）：** 调用一个或多个外部工具来执行动作。这里的工具，就可以是 Function Calling 所调用的函数。
>3. **观察（Observe）：** 接收工具执行后返回的结果。
>4. **循环：** 基于观察到的结果，再次进行思考，决定下一步行动，直到最终完成任务并向用户呈现最终结果。
>
>**本质：** 一个**自主决策和执行的闭环系统**。它的核心是“循环”和“目标导向”。
